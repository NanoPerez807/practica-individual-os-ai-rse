# Individual Assignment for Open Science and AI in Research Software Engineering
[![DOI](https://zenodo.org/badge/1161894099.svg)](https://doi.org/10.5281/zenodo.18962961)

This project analyzes 10 open‑access scientific articles obtained from arXiv, which were later processed with Grobid to generate XML files used as input for the scripts.

# Description of the scripts

## 1. Figure counter

Script: `figures.py`  
Counts `<figure>` occurrences in each XML and generates a bar chart.

The result is displayed through a bar chart showing the number of figures per article.

Generated output:

```
resultados/figuras.png
```

---

## 2. Keyword word cloud

Script: `keywords.py`

For each XML file, the content of the `<abstract>` tag is extracted, and the frequencies of each word are counted to generate a word cloud that shows the most relevant words from the abstract.

Generated outputs:

```
resultados/nube_<nombre_del_articulo>.png
```

---

## 3. Link listing

Script: `links.py`

This script searches for links inside the XML files and lists them.

Generated output:

```
resultados/enlaces.txt
```

# Installation and execution

Download the repository with

```
git clone https://github.com/NanoPerez807/practica-individual-os-ai-rse
```

The project can be run in two ways: using Poetry (local environment) or using Docker (containerized environment). Both options are explained below.

## Option 1: Using Poetry (local environment)

This option allows you to run the scripts directly on your machine using a virtual environment managed with Poetry.

### 1. Install Poetry

Follow the official instructions:

https://python-poetry.org/docs/#installation

### 2. Install dependencies

From the project root, run:

```
poetry install
```

### 3. Activate the environment

From the project root, run:

* If Poetry version is >= 2.0.0 (check with poetry --version), run:
  
```
source $(poetry env info --path)/bin/activate
```

* If Poetry version is < 2.0.0 (check with poetry --version), run:
```
poetry shell
```

### 4. Run the scripts

From the project root, run:

```
cd scripts
python figures.py
python keywords.py
python links.py
cd ..
```

### 5. Run the unit tests

From the project root, run:

```
cd tests
python -m unittest discover
cd ..
```

### 6. To exit the Poetry virtual environment when finished:

```
exit
```

---

## Option 2: Using Docker

This option allows you to run the scripts inside a Docker container without installing dependencies locally.

### 1. Install Docker

Follow the official instructions for your operating system:

https://docs.docker.com/get-docker/

### 2. Build the image

From the project root, run:

```
docker compose up --build
```

### When finished, stop and remove containers.

```
docker compose down
```

# Validation of results

Manual checks were performed to verify that the results generated by the scripts are correct.

## Keywords

The word clouds were compared manually with the content of each `<abstract>` to ensure that the most frequent terms match those in the summary.

## Figures

The number of figures was verified by manually searching for the `<figure>` tag in each XML file.

## Links

Extracted links were compared with those present in the text and XML attributes.  
Some links generated automatically by Grobid (not part of the original PDF) were excluded. Specifically, the following were discarded:

- `https://github.com/kermitt2/grobid`
- `https://raw.githubusercontent.com/kermitt2/grobid/master/grobid-home/schemas/xsd/Grobid.xsd`
- `http://www.tei-c.org/ns/1.0`
- `http://www.w3.org/2001/XMLSchema-instance`
- `http://www.w3.org/1999/xlink`

# Limitations

Some limitations arise from the challenges of programming without full mastery of the language:

* The figure count is based on occurrences of the `<figure>` tag and may include elements that are not actual visible figures.
* Abstract extraction depends on the presence of a correctly formatted `<abstract>` tag.
* The list of stopwords used for generating the word clouds is limited.
* Link extraction relies on regular expressions, so uncommon formats may not be detected.
